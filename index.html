React-Native的环境搭建(一)

发表于 2017-04-07 | 分类于 react-native |

nvm：
nodeJs版本管理工具,管理nodejs版本和npm版本

nodeJs： reactNative开发过程中所需要的代码库。

npm：是随同NodeJS一起安装的包管理工具，能解决NodeJS代码部署上的很多问题，常见的使用场景有以下几种：

  1. 允许用户从NPM服务器下载别人编写的第三方包到本地使用。
  2. 允许用户从NPM服务器下载并安装别人编写的命令行程序到本地使用。
  3. 允许用户将自己编写的包或命令行程序上传到NPM服务器供别人使用。

npm同样也可以设置淘宝镜像，让我们在下载大神jar包的时候更快捷。
npm config set registry https://registry.npm.taobao.org

NVM、NODEJS、NPM的关系

  1. nvm管理构建nodejs和对应的npm,npm管理对应nodejs的第三方插件
  2. 一个reactNative项目只包含一个nodejs和npm,npm和nodejs是配套关系， 一对一的关系。

nvm安装

nvm下载地址：https://github.com/coreybutler/nvm-windows/releases

默认安装就好，一般的默认安装路径是在C:\Users\Administrator\AppData\Roaming\nvm

nvm配置淘宝镜像 修改settings.txt,设置淘宝镜像是为了下载noods和npm速度更快。

noodjs npm安装

在确认nvm安装完成的情况下来安装noodjs和npm，以下操作都需要在电脑终端里完成

安装：nvm install 6.9.2

本地机器可以安装多个版本的noodjs，安装的时候只需要在后边更改版本号就行了。

查看已经安装的node列表：nvm list

在查看完本地安装的noodjs版本之后，我们需要选择使用版本

使用：nvm use 6.9.2

同样，在选择的时候，只要更改版本号就行了。

npm设置淘宝镜像（下载更快）

npm config set registry https://registry.npm.taobao.org
