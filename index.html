java io的使用之RandomAccessFile（六）

发表于 2017-02-10 | 分类于 java |

胡编乱造，以下就是我的理解。

这个东西我也没用过，第一次接触，网上看了看，可以用于多线程的上传和下载，下边是我自己的理解。

RandomAccessFile，别慌，当我们看到这个单词的时候，先去看下什么意思。
Random：随机
Access：访问
File：文件
随机访问文件，看上去英文是这个意思，然后我们点击查看他的源代码看下他的官方解释，英文水平有限，直接上百度了，看图。

左侧是官方的代码注释，右侧为翻译结果，大概意思是说我们可以自定义访问文件的某一区间，或者修改某一区间的内容。看图↓↓↓↓↓

这条虚线代表着一个文件，我们将他分成5个区间来读取/写入。
首先我们需要一个线程来执行写入，创建一个类继承Thread，实现构造函数，重写run方法。

RandomAccessFile写入某一区间流

public class WriteFile extends Thread {
//读取/写入的文件
File file;
//读取/写入的区间
int block;
//每个区间的长度，这个区间因文件大小而定，尽量避免浪费不必要的空间
int L = 100;

public WriteFile(File file, int block) {
    this.file = file;
    this.block = block;
}

@Override
public void run() {
    super.run();
    try {
        @SuppressWarnings("resource")
        RandomAccessFile raf = new RandomAccessFile(file, "rw");
        raf.seek((block - 1) * L);
        raf.writeBytes("this is block" + block);
        raf.writeBytes("\n");
    } catch (FileNotFoundException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    } catch (IOException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    }
}

完成以上一套流程之后，我们来看run方法应该要执行什么。创建
RandomAccessFile对象，参数第一个为读取/写入的文件名，第二个是当前文件的执行权限，参数值有这么几个（”r”[只读]，”rw”[读写]，”rws”，”rwd”），后俩不知道啥意思，前俩就够用了。
seek()方法用来指定我们要读取/写入的区间。writeBytes()用来写入内容，参数为字符串还有好多写入的方法，针对不同的需求采用不同的写入方法就行。写入的线程写好以后，我们要在主方法中运行线程，

static File file = new File("test.txt");

public static void main(String[] args) {
    if (file.exists()) {
        file.delete();
    }
    new WriteFile(file, 1).start();
    new WriteFile(file, 2).start();
    new WriteFile(file, 3).start();
    new WriteFile(file, 4).start();
    new WriteFile(file, 5).start();
    }
}

同时开启5个线程去写入文件，分别写入到不同的区间，运行查看输出。

以上是文件的大小，可以看到413个字节，细细想想我们就会知道，我们定义的每个区间大小为100个字节，如下定义↓↓↓

int L = 100;

也就是说我们一共new了5个区间，在写入数据的时候，即使填不满整个区间，而还有后续区间的时候，即使填不满，这些空间也是会为当前区间保留的。而当到了第五个区间的时候，没有后续的操作区间，他只有13个字节，写入就等于写完了。所以文件大小就是413了。

RandomAccessFile读取某一区间流

try {
    RandomAccessFile raf = new RandomAccessFile(file, "r");
    raf.seek(300);
    byte[] str = new byte[20];
    raf.read(str);
    System.out.println(new String(str));
} catch (FileNotFoundException e) {
    // TODO Auto-generated catch block
    e.printStackTrace();
} catch (IOException e) {
    // TODO Auto-generated catch block
    e.printStackTrace();
}

以上代码用来读取文件内容，可以看到RandomAccessFile()的第二个参数传的事”r”，意味着当前对象只能读，不能写。seek()移动到某一区间，这块参数传的300，也就是说从字节300开始读取。

java io的使用之纯文本文件的读取和写入（五）

发表于 2017-02-10 | 分类于 java |

胡编乱造，以下就是我的理解。

纯文本文件读取我们使用的是FileReader,通过查看源码我们发现他的父类是InputStreamReader，他的介绍也是为了方便读取字符文件，还记得我们读取char字符流的时候会用到InputStreamReader，可以联想到，其实FileReader就是对我们字符流读取和写入的封装。也就是说我们可以把
字符流的代码换成FileReader就可以了。(如图)

以下是完整代码，运行发现没有问题，就是这么搞。

public static void main(String[] args) {
    try {
        //文件读取
        FileReader fr = new FileReader("java.txt");
        BufferedReader br = new BufferedReader(fr);

        FileWriter fw = new FileWriter("java_new_file.txt");
        BufferedWriter bw = new BufferedWriter(fw);

        String input = "";
        while ((input = br.readLine()) != null) {
            bw.write(input + "\n");

        }
        bw.flush();
        bw.close();
        fw.close();
        br.close();
        fr.close();
        System.out.println("拷贝完成");
    } catch (FileNotFoundException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    } catch (IOException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    }
}

public static void main(String[] args) {

        File file = new File("java.txt");

        InputStreamReader isr = new InputStreamReader(fis, "UTF-8");

        File new_file = new File("java_new_buffer.txt");
        FileOutputStream fos = new FileOutputStream(new_file);
        OutputStreamWriter osw = new OutputStreamWriter(fos, "UTF-8");

        BufferedReader br = new BufferedReader(isr);

        osw.close();
        fos.close();
        isr.close();
        fis.close();

        System.out.println("拷贝完成");

java io的使用之字符流的读取和写入（三）

发表于 2017-02-10 | 分类于 java |

胡编乱造，以下就是我的理解，看注释

public static void main(String[] args) {
    try {
        // 读取文件对象
        File file = new File("java.txt");
        // 字节流读取
        FileInputStream fis = new FileInputStream(file);
        // 将字节流转换成字符流对象，编码格式默认为GBK，修改为UTF-8
        InputStreamReader isr = new InputStreamReader(fis, "UTF-8");

        // 写入文件对象
        File new_file = new File("java_new.txt");
        // 字节流写入
        FileOutputStream fos = new FileOutputStream(new_file);
        // 将字节流转换成字符流对象，编码格式默认为GBK，修改为UTF-8
        OutputStreamWriter osw = new OutputStreamWriter(fos, "UTF-8");

        // 字符流的读取，我们当然要用字符数组来存储了，同样的字符数组的大小也是因文件大小而定，多次调试，来选择合适的大小可以提高读取和写入效率
        char[] input = new char[100];
        int l = 0;
        //此处用while的意思是因为我们要读取的文件太大，而每回的数组存储有限，所以需要用while循环来多次读取。我们可以把鼠标放到read()的方法上看到他的返回值是一个int，然后后边介绍说，如果读取返回为-1的时候说明读取完成，根据这个我们可以写出下边的代码
        while ((l = isr.read(input)) != -1) {
            //我们为什么要用这个构造呢，因为我们并不清楚要读的字符到底有多少，而每回读取的字符数量又是固定的，为了避免浪费不必要的内存，我们采用以下构造。
            //参数含义1字符数组       2开始写入字符的偏移量      3写入的字符长度
            osw.write(input, 0, l);
        }

        //关闭流，先打开后关闭，后打卡先关闭
        osw.close();
        fos.close();
        isr.close();
        fos.close();

        System.out.println("拷贝完成");
    } catch (FileNotFoundException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    } catch (UnsupportedEncodingException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    } catch (IOException e) {
        // TODO Auto-generated catch block
        e.printStackTrace();
    }
}
